# Related Work Matrix

This matrix is a paper v0.1 positioning aid. Rows marked `citation_required`
must be externally verified in M8 before paper v0.2.

| Work | Category | What it solves | Relation to ASIEP | What ASIEP does not claim | Gap ASIEP targets |
| --- | --- | --- | --- | --- | --- |
| Self-Refine | agent self-improvement | Iterative self-feedback and revision loops. citation_required | ASIEP can record evidence about such loops when they produce a change event. | ASIEP does not provide the refinement algorithm. | Verifiable evidence object for the change, gate, and artifacts. |
| Reflexion | agent self-improvement | Memory and reflection for improving future behavior. citation_required | ASIEP can package the evidence surrounding a proposed improvement. | ASIEP does not define reflective memory or learning policy. | Agent-readable accountability for an improvement cycle. |
| Voyager | agent learning system | Long-running skill acquisition in an embodied setting. citation_required | ASIEP is narrower and can document auditable release-like changes. | ASIEP does not implement exploration or skill libraries. | Evidence profile independent of the agent runtime. |
| AgentDevel | release engineering for agents | Development and release practices for agents. citation_required | ASIEP is complementary: it supplies evidence objects and validation. | ASIEP does not replace release engineering workflow. | Machine-checkable evidence layer for review. |
| AuditableLLM | auditability and logging | Audit-oriented records for language-model systems. citation_required | ASIEP shares the auditability goal but focuses on self-improvement evidence. | ASIEP does not claim coverage of all LLM audit cases. | Minimal profile, repair plan, bundle resolver, and package chain. |
| OpenTelemetry GenAI | tracing | GenAI-oriented spans and attributes. citation_required | M4 uses local OTel-like fixtures as trace sources. | ASIEP does not call a collector or implement the whole ecosystem. | Bridge from trace source to accountability evidence. |
| LangSmith | tracing and feedback | Trace, run, thread, and feedback concepts. citation_required | M4 uses local LangSmith-like fixtures as source shape examples. | ASIEP does not call the real service or replace its tracing model. | Evidence profile that can receive trace-derived artifacts. |
| W3C PROV | provenance | Activities, agents, and entities. citation_required | ASIEP maps cycle, agent, blueprint, and artifact relations to PROV-like JSON-LD. | ASIEP does not replace PROV semantics. | Domain profile for self-improvement evidence and gates. |
| FDO | digital objects | Persistent object concepts and typed operations. citation_required | M5 emits local FDO-like records with local identifiers and operations. | ASIEP does not register with an external FDO service. | Local object record for agent package discovery. |
| RO-Crate / Workflow Run RO-Crate | research object packaging | JSON-LD package metadata for data and workflow runs. citation_required | M5 emits local RO-Crate-like metadata for evidence packages. | ASIEP does not assert community validation. | Accountability-focused metadata over ASIEP bundles and artifacts. |
| EU AI Act / AI governance logging | governance | Accountability and logging requirements for AI systems. citation_required | ASIEP can provide machine-readable evidence inputs for governance review. | ASIEP does not provide legal compliance by itself. | Concrete evidence protocol for agent change events. |
| NIST AI RMF / AI Agent Standards Initiative | risk management and standards | Vocabulary and risk-management framing. citation_required | ASIEP can supply technical artifacts for risk review. | ASIEP does not claim formal alignment or adoption. | Agent-readable evidence that can be inspected and tested. |
| OWASP GenAI / agentic risks | security risks | Risk catalogues for GenAI and agentic systems. citation_required | ASIEP attack samples cover a small subset of evidence tampering risks. | ASIEP does not cover every agentic risk. | Evidence tamper detection and repair boundary documentation. |
| SLSA / OpenSSF provenance | software supply-chain provenance | Build provenance and integrity practices. citation_required | ASIEP borrows the idea that artifacts and decisions need verifiable provenance. | ASIEP does not implement supply-chain build guarantees. | Self-improvement evidence provenance for agent changes. |
