# Reviewer Attack-Defense Matrix

| Attack | Why it is partially valid | Defense | Evidence in repository | Limitation to acknowledge | Future strengthening |
| --- | --- | --- | --- | --- | --- |
| This is just another JSON schema. | Schema alone would not be a research contribution. | ASIEP includes validator, repairer, resolver, importer, packager, evaluator, corpus, and paper claim linting. | `src/`, `interfaces/`, `evaluation/`, `reports/` | Current examples are local. | Add independent implementation and external traces. |
| AgentDevel already solves this. | Release engineering overlaps with self-improvement review. | ASIEP supplies a profile-level evidence object and machine checks, not a development workflow. | `manuscript/related_work_matrix.md`, `profiles/asiep/v0.1/profile.json` | External related-work claims need M8 citation hardening. | Add direct comparison after verifying AgentDevel details. |
| OpenTelemetry already records traces. | Traces are a major input source. | ASIEP turns trace-derived artifacts into evidence refs, gates, digest checks, repair plans, and packages. | `src/asiep_importer/`, `examples/fixtures/`, `docs/trace_importer.md` | No real collector is called. | Add importer for exported trace files after M8. |
| W3C PROV already provides provenance. | PROV covers activities, agents, and entities. | ASIEP is a domain profile that adds gates, safety checks, flip thresholds, repairability, and bundle validation. | `mappings/prov_mapping.md`, `evaluation/crosswalk/asiep_v0.1_crosswalk.json` | PROV mapping is local and minimal. | Validate JSON-LD against richer provenance tooling. |
| RO-Crate / FDO already package evidence. | Package ecosystems already exist. | ASIEP uses local package metadata to make self-improvement evidence exchangeable and reviewable. | `src/asiep_packager/`, `interfaces/asiep_package_manifest.schema.json` | No external registry or community validation. | Test community tooling and registry workflows later. |
| Evaluation uses only local fixtures. | The evaluation corpus is small and synthetic. | M6 is designed as reproducible v0.1 evidence, not a broad benchmark. | `evaluation/corpus/asiep_v0.1_corpus.json`, `reports/asiep_v0.1_evaluation_report.json` | Results should not be generalized. | Add real trace exports and larger adversarial corpora. |
| Metrics are too optimistic. | Values like 1.0 can be overread. | The report includes numerator, denominator, method, and limitations for each metric. | `reports/asiep_v0.1_evaluation_report.json`, `paper_assets/tables/evaluation_metrics.md` | Metrics are local regression indicators. | Add harder corpora and confidence intervals. |
| Privacy compliance is not real DLP. | Sentinel/key scanning is not privacy enforcement. | The paper states this limitation and the importer blocks the provided sensitive fixture. | `src/asiep_evaluator/evaluator.py`, `manuscript/limitations.md` | No production privacy system is present. | Integrate policy-aware redaction and DLP checks later. |
| No real OTel / LangSmith integration. | Local fixtures do not prove external integration. | M4 is explicitly a local trace-to-evidence bridge. | `examples/fixtures/`, `docs/trace_importer.md` | No external API call is made. | Add verified adapters after external reference hardening. |
| No real FDO registry or PID. | Local identifiers are not persistent registry identifiers. | M5 intentionally avoids fake registry claims. | `docs/fdo_rocrate_packaging.md`, `profiles/asiep/v0.1/package_policy.json` | Package identifiers are local. | Add optional registry submission only after policy review. |
| Repairer may encourage evidence tampering. | A repair plan can be misused if it edits evidence to pass. | M2 repair policy forbids forged gate reports, digests, approvals, and safety-result falsification. | `profiles/asiep/v0.1/repair_policy.json`, `src/asiep_repairer/repairer.py` | Human review is still needed for external evidence. | Add signed evidence and review workflow. |
| ASIEP may be too narrow because it focuses on self-improvement. | The profile does not cover all agent governance events. | Narrowness is a v0.1 design choice to make validation and evaluation concrete. | `schemas/asiep.schema.json`, `manuscript/limitations.md` | Generalization is not proven. | Define additional AAEL profiles after ASIEP stabilizes. |
