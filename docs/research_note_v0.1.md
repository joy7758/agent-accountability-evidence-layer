# Research Note v0.1

AAEL focuses on accountability evidence for agent self-improvement claims.

The project deliberately avoids building a more autonomous agent. In an
agent-to-agent environment, the durable layer is not another runtime loop; it is
a compact evidence profile that independent agents can validate, compare, and
map into adjacent standards.

## v0.1 Scope

ASIEP v0.1 contains:

- schema-level constraints
- JSON-LD terms
- a lifecycle state machine
- ten invariants
- a local validator
- positive and negative examples
- mapping notes for PROV-O, OpenTelemetry, FDO, and RO-Crate

## Non-Goals

- no model invocation
- no self-modifying agent runtime
- no commercial customer support demo
- no deployment automation
- no importer/exporter beyond mapping notes

## Why Evidence First

Self-improvement claims are easy to overstate. A minimal evidence profile makes
the claim inspectable by another agent before the claim becomes policy, code, or
runtime behavior. This shifts the project toward verifiable accountability
instead of opaque automation.
