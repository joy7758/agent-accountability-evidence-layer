# Repository Instructions

Future programming in this repository should be designed around AI agents, not
only traditional human-computer interaction. Code and documentation should make
agent self-inspection, evidence checking, and iterative improvement easier.

For ASIEP v0.1:

- Preserve the narrow scope: schema, JSON-LD context, state machine,
  invariants, validator, attack samples, and mappings.
- Do not introduce large language model calls.
- Do not introduce a commercial customer-service application.
- Do not introduce automated deployment.
- Prefer small consistency fixes over broad feature expansion.
- Keep machine-readable evidence, validator codes, and docs aligned so future
  agents can reason over the repository without relying on hidden assumptions.
